Elementos, Componentes e Configuração de Rede Interna

Essa seção aborda a configuração básica de redes internas usando o Cisco Packet Tracer. O Packet Tracer é uma ferramenta de simulação que permite criar, visualizar e configurar redes de forma prática.

Entre os componentes configuráveis, temos:

  • Roteadores e switches: Interligam segmentos de rede e permitem a comunicação entre diferentes dispositivos.
  • Endereçamento IP: Configuração de IPs estáticos ou dinâmicos para dispositivos em uma rede.
  • Topologia: Estrutura física ou lógica que define como os dispositivos estão conectados.

Configuração dos Servidores

Aqui discutimos os servidores fundamentais para o funcionamento de redes:

  • Servidor DNS: Resolve nomes de domínio (ex.: www.exemplo.com) para endereços IP, facilitando a navegação na web.
  • Servidor HTTP: Responsável pela hospedagem de páginas e comunicação web. É o servidor que lida com requisições de navegação em sites.
  • Servidor DHCP: Distribui automaticamente endereços IP para dispositivos na rede, simplificando a configuração de rede para usuários.

Equipamentos em Redes

Essa seção explora os principais equipamentos usados em redes:

  • Switch: Dispositivo que conecta vários dispositivos em uma rede local (LAN) e direciona dados para o dispositivo correto.
  • Roteador: Conecta redes diferentes e encaminha pacotes de dados entre elas. É essencial para acesso à internet.
  • Ponto de Acesso (Access Point): Permite a conexão de dispositivos sem fio à rede.
  • Firewall: Controla o tráfego de entrada e saída na rede, protegendo contra acessos não autorizados.

Conceito de Servidor

Um servidor é um dispositivo ou programa que fornece recursos, dados e serviços a outros dispositivos (chamados clientes) na rede. Ele desempenha diversas funções, como armazenar arquivos, hospedar websites e fornecer acesso a bancos de dados.

Servidores podem ser físicos ou virtuais e são fundamentais para o funcionamento de redes corporativas e da internet.

DNS - O que é

O DNS (Domain Name System) é como a "lista telefônica" da internet. Ele converte nomes de domínio amigáveis (ex.: www.site.com) em endereços IP numéricos que computadores entendem. Isso permite uma navegação mais simples para o usuário.

HTTP - O que é

O HTTP (Hypertext Transfer Protocol) é o protocolo usado para comunicação entre navegadores e servidores. Ele define como as mensagens são formatadas e transmitidas e quais ações os servidores e navegadores devem tomar em resposta a diferentes comandos.

DHCP - O que é

O DHCP (Dynamic Host Configuration Protocol) é um protocolo que automaticamente fornece um endereço IP e outras configurações de rede para dispositivos em uma rede. Ele simplifica o processo de configuração, eliminando a necessidade de atribuição manual de IPs.

Foto da Crimpagem

A crimpagem é o processo de fixação de um conector (RJ-45) em um cabo de rede. É essencial para garantir que a conexão entre dispositivos seja estável e sem interferências.

Cabo de Rede - Pinagem

A pinagem refere-se à ordem dos fios no conector RJ-45, que deve seguir padrões específicos (como T568A ou T568B) para garantir o funcionamento adequado. O padrão define a posição de cada fio de acordo com sua cor.

Modelo OSI

O modelo OSI (Open Systems Interconnection) é uma estrutura em camadas que descreve o processo de comunicação entre sistemas de rede. Ele possui 7 camadas, cada uma com funções específicas, desde a camada física até a aplicação.

Camadas do modelo OSI:

  • 1 - Física
  • 2 - Enlace
  • 3 - Rede
  • 4 - Transporte
  • 5 - Sessão
  • 6 - Apresentação
  • 7 - Aplicação

Conceito de TCP/IP

O modelo TCP/IP é uma arquitetura de rede composta por quatro camadas e é a base da internet. Ele define como os dados são transmitidos e roteados de um dispositivo a outro.

Camadas do modelo TCP/IP:

  • 1 - Acesso à Rede
  • 2 - Internet
  • 3 - Transporte
  • 4 - Aplicação
